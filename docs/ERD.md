# StudyCircle Database Documentation

## Overview

StudyCircle uses PostgreSQL as the primary database. The schema consists of 9 core entities designed to support secure authentication, study group collaboration, scheduling, material sharing, detailed session attendance tracking, and dynamic learning progress analysis.

---

## Entity Specifications

### USERS

**Purpose:**  
Store user account credentials, basic profile data, and system-level roles.

**Fields:**

* `id` (UUID, Primary Key)
* `username` (VARCHAR, Unique, Indexed)
* `email` (VARCHAR, Unique, Indexed)
* `password_hash` (VARCHAR)
* `full_name` (VARCHAR)
* `semester` (INTEGER, Optional)
* `role` (VARCHAR, Default: 'student') - *System-wide role*
* `created_at` (TIMESTAMP, Default: NOW())
* `updated_at` (TIMESTAMP, Default: NOW())

**Relationships:**

* One-to-One with `LEARNING_STYLES`
* One-to-Many with `MEMBERS` (membership within study groups)
* One-to-Many with `MATERIALS` (materials uploaded by this user)
* One-to-Many with `PROGRESS` (progress tracker across groups)
* One-to-Many with `SESSION_ATTENDANCES` (session participation logs)

---

### LEARNING_STYLES

**Purpose:**  
Store user learning preferences to assist in future smart matching features.

**Fields:**

* `id` (UUID, Primary Key)
* `user_id` (UUID, Foreign Key → USERS.id, Unique, Indexed)
* `primary_style` (VARCHAR) - *e.g., Visual, Auditory, Reading/Writing, Kinesthetic*
* `secondary_style` (VARCHAR, Optional)

**Relationships:**

* One-to-One with `USERS` (Cascade delete on user removal)

---

### SUBJECTS

**Purpose:**  
Store official academic subject catalogs to categorize study groups.

**Fields:**

* `id` (UUID, Primary Key)
* `code` (VARCHAR, Unique, Indexed) - *e.g., IF-201*
* `name` (VARCHAR) - *e.g., Advanced Web Programming*
* `description` (TEXT, Optional)

**Relationships:**

* One-to-Many with `STUDY_GROUPS`

---

### STUDY_GROUPS

**Purpose:**  
Store study groups generated by users.

**Fields:**

* `id` (UUID, Primary Key)
* `subject_id` (UUID, Foreign Key → SUBJECTS.id, Indexed)
* `created_by` (UUID, Foreign Key → USERS.id)
* `name` (VARCHAR, Indexed)
* `description` (TEXT, Optional)
* `max_members` (INTEGER, Default: 10)
* `created_at` (TIMESTAMP, Default: NOW())

**Relationships:**

* Many-to-One with `SUBJECTS`
* Many-to-One with `USERS` (Creator)
* One-to-Many with `MEMBERS`
* One-to-Many with `SESSIONS`
* One-to-Many with `MATERIALS`
* One-to-Many with `PROGRESS`

---

### MEMBERS

**Purpose:**  
Track study group memberships and establish group-level permissions (Group Admin vs Regular Member).

**Fields:**

* `id` (UUID, Primary Key)
* `study_group_id` (UUID, Foreign Key → STUDY_GROUPS.id, Indexed)
* `user_id` (UUID, Foreign Key → USERS.id, Indexed)
* `role` (VARCHAR, Default: 'member') - *Values: 'admin' (Group Admin) or 'member'*
* `joined_at` (TIMESTAMP, Default: NOW())

**Constraints:**

* Unique Constraint: `(study_group_id, user_id)` - *A user can only join a group once.*

**Relationships:**

* Many-to-One with `STUDY_GROUPS`
* Many-to-One with `USERS`

---

### SESSIONS

**Purpose:**  
Store scheduled learning activity structures within study groups.

**Fields:**

* `id` (UUID, Primary Key)
* `study_group_id` (UUID, Foreign Key → STUDY_GROUPS.id, Indexed)
* `created_by` (UUID, Foreign Key → USERS.id)
* `title` (VARCHAR)
* `scheduled_start_time` (TIMESTAMP)
* `scheduled_end_time` (TIMESTAMP)
* `status` (VARCHAR, Default: 'scheduled') - *Values: 'scheduled', 'active', 'completed', 'cancelled'*
* `created_at` (TIMESTAMP, Default: NOW())

**Relationships:**

* Many-to-One with `STUDY_GROUPS`
* Many-to-One with `USERS` (Creator)
* One-to-Many with `SESSION_ATTENDANCES`

---

### SESSION_ATTENDANCES

**Purpose:**  
Track exact study session participation details for each student, driving the automatic progress computation system.

**Fields:**

* `id` (UUID, Primary Key)
* `session_id` (UUID, Foreign Key → SESSIONS.id, Indexed)
* `user_id` (UUID, Foreign Key → USERS.id, Indexed)
* `joined_at` (TIMESTAMP, Default: NOW())
* `left_at` (TIMESTAMP, Optional)
* `duration_minutes` (INTEGER, Optional)
* `status` (VARCHAR, Default: 'active') - *Values: 'active', 'completed', 'left_early'*
* `created_at` (TIMESTAMP, Default: NOW())

**Constraints:**

* Unique Constraint: `(session_id, user_id)` - *One attendance record per session per user.*

**Relationships:**

* Many-to-One with `SESSIONS`
* Many-to-One with `USERS`

---

### MATERIALS

**Purpose:**  
Store metadata and references for shared educational resources.

**Fields:**

* `id` (UUID, Primary Key)
* `study_group_id` (UUID, Foreign Key → STUDY_GROUPS.id, Indexed)
* `uploaded_by` (UUID, Foreign Key → USERS.id)
* `title` (VARCHAR)
* `file_url` (VARCHAR)
* `file_type` (VARCHAR, Optional) - *e.g., pdf, zip, pptx*
* `file_size` (INTEGER, Optional) - *Size in bytes*
* `uploaded_at` (TIMESTAMP, Default: NOW())

**Relationships:**

* Many-to-One with `STUDY_GROUPS`
* Many-to-One with `USERS` (Uploader)

---

### PROGRESS

**Purpose:**  
Aggregate and track study consistency, total session times, and estimated mastery levels per student per group.

**Fields:**

* `id` (UUID, Primary Key)
* `user_id` (UUID, Foreign Key → USERS.id, Indexed)
* `study_group_id` (UUID, Foreign Key → STUDY_GROUPS.id, Indexed)
* `subject_id` (UUID, Foreign Key → SUBJECTS.id)
* `sessions_attended` (INTEGER, Default: 0)
* `total_study_hours` (DOUBLE PRECISION, Default: 0.0)
* `estimated_mastery_level` (VARCHAR, Default: 'Beginner') - *Values: 'Beginner' (0-4h), 'Intermediate' (5-14h), 'Advanced' (15h+)*
* `updated_at` (TIMESTAMP, Default: NOW())

**Constraints:**

* Unique Constraint: `(user_id, study_group_id)` - *One aggregated progress sheet per group membership.*

**Relationships:**

* Many-to-One with `USERS`
* Many-to-One with `STUDY_GROUPS`
